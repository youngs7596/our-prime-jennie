# 🗄️ 장기 데이터 축적 전략 (최종 v1.0)
**목표**: 시스템의 "기억"을 구축하여 단순 봇에서 생각하는 파트너로 진화

## 1. 핵심 철학
**"훌륭한 시스템은 얼마나 성공했는지가 아니라, *왜* 행동하지 않았는지를 기억하는가로 정의된다."** (Junho)
우리는 **결정**, **컨텍스트**, 그리고 **놓친 기회**를 수집합니다.

## 2. Phase 0: 인프라 (Minji의 기반)
수집 전에 먼저 지속성을 보장해야 합니다.
*   **스키마 버전 관리**: 모든 로그에 `schema_v` 태그 필수
*   **시간 표준**: 모든 타임스탬프는 **UTC** (ISO 8601)
*   **저장소**: Raw (JSON/Text) vs Processed (SQL/Parquet) 분리
*   **백업**: 주간 자동 백업을 외부 콜드 스토리지로

## 3. Priority 1: 두뇌의 기억 (Decision Ledger) **[구현 완료]**
**지금 반드시 필요.** "진실의 순간"을 포착.

| 카테고리 | 지표 / 데이터 포인트 | 목적 |
| :--- | :--- | :--- |
| **컨텍스트** | Hunter Score, Market Regime, **주요 키워드** (벡터화) | "역사는 반복된다" (Jennie의 서사) |
| **프로세스** | **토론 로그**: "반대 포지션" 논거 (리스크 vs 기회) | *우리의* 변증법으로 미래 로직 미세조정 |
| **운영 데이터** | `thinking_called` (bool), `thinking_reason`, `cost_estimate`, `gate_result` | "결정 비용" 분석 (Junho의 운영) |
| **결과** | 분류: 매수 / 매도 / 보유 / **No-Decision** (보류)<br>이유: 한 줄 로직 | "비겁함" vs "규율" 구분 |
| **성과** | T+5, T+20 수익률, MDD | "정답지" |

## 4. Priority 2: "Shadow Radar" (놓친 기회) **[구현 완료]**
**전략적 가치.** 필터링된 것을 포착하여 리스크 민감도 보정.
*   **트리거**:
    *   가격 급등 (+15%)
    *   **거래량 쇼크** (상위 X% 변동)
    *   **갭 상승/하락** 이벤트
    *   **변동성 돌파**
*   **데이터**: 해당 시점의 Hunter Score, 거부 이유
*   **목표**: "우리가 너무 소심한가?" 질문에 답하기

## 5. Priority 2.5: 수급 데이터 (트렌드 기초) **[구현 완료]**
**Minji의 요청.** 시장의 "조류".
*   **일간**: 외국인 순매수, 기관 순매수, 프로그램 순매수
*   **패턴**: 예: "외국인 순매수 3일 연속"

## 6. Priority 3: 타겟팅된 장중 데이터 **[구현 완료]**
**최적화 가치.** **Radar 후보**에 대해서만.
*   **데이터**: 1분봉 OHLCV 및 VWAP
*   **수급**: **외국인 추정 순매수**, **프로그램 순매수** (1분)
*   **목표**: "진입 타이밍" 분석 및 "가짜" 움직임 구분 (Jennie의 K-Stock 팁)

## 7. Priority 4: 시장 국면 메타데이터
**컨텍스트 가치.**
*   **데이터**: KOSPI 변동성, 원/달러, 미국 금리, VIX, 공매도 비중, 신용 잔고
*   **목표**: "모드 전환" 자동화 (공격적 vs 방어적)

## 8. 구현 로드맵
1.  **Phase 0 (설정)**: 스키마 정의 (SQL/JSON) 및 버전 관리
2.  **Phase 1 (두뇌)**: `DecisionContext` 객체를 통해 `llm_decision_ledger` 구현
3.  **Phase 2 (그림자)**: Scout에서 `ShadowRadar` 로직 구축
4.  **Phase 3 (수급)**: 장중 투자자 수급 수집하도록 `news-crawler` 또는 collector 업데이트
5.  **Phase 4 (분석)**: "실패 분석 대시보드" 구축
