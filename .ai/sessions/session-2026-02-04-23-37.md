# Session Handoff: RabbitMQ CPU 폭주 해결 및 장비 안정화

## 요약 (Summary)
WSL 환경에서 `beam.smp`(RabbitMQ)가 CPU를 과도하게 점유하는 문제를 해결하고 시스템을 안정화했습니다. 배당 전략(Dividend Capture) 백테스트를 위한 준비 작업을 마쳤습니다.

## 변경 사항 (Changes)
- `docker-compose.yml`: RabbitMQ 에 Erlang 스케줄러 최적화 옵션(`-+sbwt none`) 추가하여 CPU 점유율 감소 (990% -> 0.1%).
