# Session Handoff: Backfill LLM Gateway 안정화

## 요약 (Summary)
- 백테스트/백필 파이프라인의 데이터 공백 및 과도한 손실 원인을 점검하고, 일중 경로/유니버스/뉴스 컷오프 등 현실화 개선을 적용했습니다.
- KOSPI 지수 데이터 누락을 보완하기 위한 백필 스크립트를 추가했고, WATCHLIST_HISTORY 백필용 스크립트를 로컬 LLM 전용으로 정리했습니다.
- Ollama Gateway(/api/chat) 경유 로컬 LLM 호출로 전환했지만, gateway 500/503 오류가 발생하여 배치/스로틀링 추가 필요성이 확인되었습니다.

## 변경 사항 (Changes)
- AGENTS.md: 운영 지침 문서 정리/갱신.
- utilities/backtest_scout_e2e.py: OHLC ZigZag 일중 경로 기본화, 뉴스 컷오프(09:00), 동적 유니버스, 거래량 기반 체결 제한/확률, 초기자본 기본값 변경.
- utilities/backtest_gpt_v2.py: 거래량 기반 체결 제한/확률, 초기자본 기본값 변경.
- utilities/backfill_kospi_index.py: KOSPI 지수(0001) 백필 스크립트 추가.
- utilities/backfill_watchlist_history_llm.py: 로컬 LLM 전용(클라우드 폴백 제거), /api/chat 경유 호출로 전환, gateway 기본 환경변수 추가.
- services/scout-job/scout.py: WATCHLIST_HISTORY 저장 경로 개선(세부 변경 반영).
