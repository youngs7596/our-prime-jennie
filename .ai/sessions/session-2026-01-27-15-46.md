# Session Handoff: Visual Logic Crash Fix

## 요약 (Summary)
"Visual Logic" 페이지의 "Real-Time Observability" 모드에서 발생하던 치명적인 크래시(White Screen) 문제를 해결했습니다. 원인은 실시간 데이터 스트림 내의 중복된 타임스탬프와 정렬되지 않은 데이터였으며, 이를 프론트엔드 레벨에서 엄격하게 검증, 중복 제거, 정렬하여 해결했습니다. 또한 보조지표(MA, BB) 렌더링 로직을 안전하게 수정하여 차트 안정성을 확보했습니다.

## 변경 사항 (Changes)
- `services/dashboard/frontend/src/pages/logic/VisualLogic.tsx`:
    - **Data Deduplication**: `Map`을 사용하여 타임스탬프 기준 중복 데이터 제거 로직 추가.
    - **Strict Validation**: `Open`, `High`, `Low`, `Close` 필드의 숫자 유효성(NaN 체크) 검사 강화.
    - **Sorting**: 시간순(`time`) 정렬 강제 적용.
    - **Safe Indicator Rendering**: 캔들 데이터와 지표 데이터의 동기화를 보장하고, 렌더링 실패 시에도 페이지가 크래시되지 않도록 예외 처리 추가.
- `docs/changelogs/CHANGELOG-2026-01.md`: 2026-01-27 작업 내역 업데이트.
