# Session Handoff: Dashboard Operations Debugging & UI Enhancement

## 요약 (Summary)
대시보드와 Airflow 간의 데이터 연동 문제(401, 데이터 비어있음, 로그 502 오류)를 해결하고, Operations 페이지의 UI(서비스 목록, 시간 범위 필터, 한국 시간 표시)를 개선했습니다.

## 변경 사항 (Changes)
- `docker-compose.yml`: `network_mode: host` 사용에 따른 포트 매핑 문제 해결 (localhost 포트 사용), Airflow Basic Auth 활성화.
- `services/dashboard/backend/routers/airflow.py`: Airflow API URL을 `localhost:8085`로 변경.
- `services/dashboard/backend/routers/logs.py`: Loki URL을 `localhost:3400`으로 변경, 쿼리 라벨을 `container_name`에서 `app`으로 수정, 서비스 목록 최신화.
- `services/dashboard/frontend/src/pages/Operations.tsx`:
  - DAG `schedule_interval` 객체 렌더링 오류 수정 (React Error #31).
  - 로그 시간 범위(5m/30m/1h/6h) 선택 버튼 UI 및 기능 구현.
  - 서비스 목록 업데이트 (`scout-worker`, `buy-scanner` 등 추가, `jenkins` 제거).
  - 로그 타임스탬프 한국 시간(KST) 표시 적용.
  - 불필요한 중복 try-catch 구문 제거 및 린트 오류 수정.
