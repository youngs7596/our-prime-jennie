# Session Handoff: Fix Portfolio Data & Stop Loss Bug

## 요약 (Summary)
`portfolio` 테이블의 데이터 정합성 문제(중복 레코드, 매수 금액 오류)를 해결하고, `buy-executor` 서비스에서 `STOP_LOSS_PRICE`가 저장되지 않던 버그를 수정했습니다. 또한 `rules.md`에 한국어 사용 강제 규칙을 추가하고 모든 문서를 한국어로 현행화했습니다.

## 변경 사항 (Changes)
- `services/buy-executor/executor.py`: 신규 매수 시 `risk_setting` 또는 기본값(-5%)을 사용하여 손절가(`stop_loss_price`)를 DB에 저장하도록 수정
- `scripts/fix_portfolio_data.py`: [신규] 현대차 중복 레코드 정리 및 `total_buy_amount` 재계산 스크립트 작성 및 실행
- `scripts/fix_stop_loss.py`: [신규] 기존 보유 종목 중 손절가가 누락된 건(미래에셋생명, SK스퀘어)에 대해 -5% 자동 설정 백필 실행
- `rules.md`: **Critical Rule** (AI 답변 한국어 강제) 추가
- `CHANGELOG.md`: 2025-12-25 작업 내역(Data Integrity, Critical Bug Fix) 업데이트
- `walkthrough.md`: 포트폴리오 수정 내역 상세 보고서 (한국어) 작성
