# Session Handoff: Implement Silent Stall Detection & Dashboard Monitor

## 요약 (Summary)
이번 세션에서는 `PriceMonitor`의 WebSocket 데이터 수신 중단 현상(Silent Stall)을 자동으로 감지하고 복구하는 로직을 추가했습니다. 또한, `OpportunityWatcher`의 실시간 상태 정보(수신 틱 수, 시그널 수 등)를 Redis를 통해 대시보드 System 페이지에서 실시간으로 모니터링할 수 있도록 통합했습니다.

## 변경 사항 (Changes)
- `services/price-monitor/monitor.py`:
  - `_monitor_with_websocket` 루프 내에 Silent Stall 감지 (60초 타임아웃) 및 재연결 로직 추가.
  - 5초마다 `opportunity_watcher.publish_heartbeat()` 호출하여 상태 전파.
- `services/price-monitor/opportunity_watcher.py`:
  - `publish_heartbeat()` 메서드 추가 (Redis Key: `monitoring:opportunity_watcher`, TTL: 15s).
- `services/dashboard/backend/routers/system.py`:
  - `GET /api/system/realtime-monitor` 엔드포인트 추가 (Redis 데이터 조회).
- `services/dashboard/frontend/src/lib/api.ts`:
  - `systemApi.getRealtimeMonitor` 함수 추가.
- `services/dashboard/frontend/src/pages/System.tsx`:
  - "Real-time Watcher Status" 카드 추가 (Hot Watchlist, Tick Count, Signals 등 시각화).
- `tests/services/price-monitor/test_monitor.py`:
  - `test_monitor_websocket_silent_stall` 단위 테스트 추가 및 검증 완료.
